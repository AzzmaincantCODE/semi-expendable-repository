# Semi-Property Guardian — Development History & Major Changes

This file documents the major features, refactoring, and configurations applied to the codebase. It serves as the primary context log for developers and AI agents.

---

## 📅 July 2026

### 🖥️ Desktop Application Packaging (Electron Integration)
*   **Change**: Integrated Electron to run the application as a standalone desktop app (`.exe`).
*   **Why**: The user wanted to open the application without running terminal commands (`npm run dev`) or using a standard web browser.
*   **Implementation**:
    *   Created `electron/main.cjs` to define window size, basic menus (File, Edit, View), and load the static index file.
    *   Modified `vite.config.ts` to dynamically resolve asset base paths relatively (`./` when in Electron mode, `/semi-property-guardian/` otherwise).
    *   Configured `electron-packager` in `package.json` with scripts to build and package the app while ignoring heavy source and node directories to avoid Windows Defender file locks (`EPERM`).
    *   Created a VBScript launcher `Semi-Property Guardian.vbs` in the project root for double-click startup.

### 📥 Auto-Backup Excel Exports
*   **Change**: Created a unified, live backup system in the main Header.
*   **Why**: The user requested that all major data tables (Inventory, ICS, ITR, Returns, Registry, etc.) support exporting to Excel as live backups.
*   **Implementation**:
    *   Created `src/utils/excelExport.ts` utilizing SheetJS (`xlsx`) to query database tables and compile them into formatted spreadsheets.
    *   Added a "📥 Backup" dropdown in the main `Header.tsx` showing spinners during active queries and saving directly to the Downloads folder.
    *   Implemented a **"Full Backup"** option which compiles all 10 major datasets (Inventory, ICS Slips, Property Cards, ITR Transfers, Purchase Orders, Physical Counts, Loss Reports, Registry Rows, Return Slips, and RSPI weekly summaries) into separate sheets of a single workbook.

### 🚨 Improved Mutation Error Toast Alerts
*   **Change**: Display specific database exception messages (such as duplicate property numbers) on screen.
*   **Why**: In legacy mode, when a user entered a duplicate property number, the error failed silently on the UI and only printed in the developer console.
*   **Implementation**:
    *   Modified `createItemMutation` and `updateItemMutation` in `src/pages/Inventory.tsx`.
    *   Ensured update mutation throws when service response success is false.
    *   Mapped `onError` to trigger a destructive React UI toast displaying the exact database/uniqueness validation error message.

## 📅 June 2026

### 🔄 Warranty & Lifespan Migration
*   **Change**: Moved the "Item Warranties & Lifespan" monitoring dashboard from the main Inventory page.
*   **Why**: The Inventory page was cluttered, and the user requested placing these features inside the lookups configuration dashboard.
*   **Implementation**:
    *   Created `src/components/lookups/WarrantiesTab.tsx` and refactored it to consume inventory items and calculate countdown thresholds via `date-fns`.
    *   Mounted it as a standalone tab in `src/pages/Lookups.tsx`.
    *   Removed the side grid column and references from `src/pages/Inventory.tsx` allowing the inventory table to take up the full screen width.

### 🛠️ Legacy Item Workflow
*   **Change**: Enabled manual entry of older items purchased before the system was active.
*   **Why**: Autogenerated property numbers (`SPLV-...` or `SPHV-...`) would override manual entries for items that already had historical paper property cards.
*   **Implementation**:
    *   Added "Add Legacy Item" buttons to `Inventory.tsx` header.
    *   Created a "Legacy Mode" checkbox toggle at the top of the compact grid in `src/components/inventory/InventoryForm.tsx` to disable automatic code generation and bypass strict SPHV/SPLV schema validation.

### 📱 Responsive Layout & Zoom Fixes
*   **Change**: Eliminated horizontal scrollbars on all pages and respected browser zoom settings.
*   **Why**: The UI overflowed on standard resolutions and forced horizontal scrolls.
*   **Implementation**:
    *   Modified `src/index.css` to add `overflow-x: hidden` and `max-width: 100vw`.
    *   Wrapped the main inventory table card in a `min-w-0` block to allow CSS Grid layout calculations to shrink gracefully.
